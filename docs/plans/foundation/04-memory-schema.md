# Phase 4: Memory Schema

## Motivation

Memory is Theo's long-term mind. Before building any memory services, the database schema must
exist. This phase creates all memory-related tables in a single migration — the foundation for the
knowledge graph, episodic memory, core memory, user model, and self model.

By landing all tables at once, later phases (5-8) can build services against a stable schema without
interleaving migrations with application code. This keeps the dependency chain clean.

## Design Decisions

### Shared Utility Functions

These triggers are generic utilities used across multiple tables. They are defined once before any
table that references them.

```sql
-- Auto-update updated_at on any row modification.
-- Attach to any table with an updated_at column.
CREATE OR REPLACE FUNCTION update_updated_at() RETURNS trigger AS $$
BEGIN
  NEW.updated_at := now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Auto-populate search_text from body using english text search config.
-- Attach to any table with body (text) and search_text (tsvector) columns.
CREATE OR REPLACE FUNCTION update_search_text() RETURNS trigger AS $$
BEGIN
  NEW.search_text := to_tsvector('english', NEW.body);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### Identity Column Strategy

All tables use `GENERATED BY DEFAULT AS IDENTITY` instead of `GENERATED ALWAYS AS IDENTITY`. Event
sourcing requires the ability to replay events and recreate rows with their original IDs. `BY
DEFAULT` allows explicit ID insertion during projection rebuilds while still auto-generating IDs for
normal inserts.

### Knowledge Graph: `node`

```sql
CREATE TABLE IF NOT EXISTS node (
  id          integer     GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  kind        text        NOT NULL
              CHECK (kind IN (
                'fact', 'preference', 'observation',
                'belief', 'goal', 'person', 'place',
                'event', 'pattern', 'principle'
              )),
  body        text        NOT NULL,
  embedding   vector(768),           -- pgvector, populated async by embedding service
  trust       text        NOT NULL DEFAULT 'inferred'
              CHECK (trust IN ('owner', 'owner_confirmed', 'verified', 'inferred', 'external', 'untrusted')),
  confidence  real        NOT NULL DEFAULT 1.0
              CHECK (confidence >= 0.0 AND confidence <= 1.0),
  importance  real        NOT NULL DEFAULT 0.5
              CHECK (importance >= 0.0 AND importance <= 1.0),
  sensitivity text        NOT NULL DEFAULT 'normal'
              CHECK (sensitivity IN ('normal', 'financial', 'medical', 'identity', 'location', 'relationship')),
  search_text tsvector,              -- populated by trigger from body
  created_at  timestamptz NOT NULL DEFAULT now(),
  updated_at  timestamptz NOT NULL DEFAULT now(),
  access_count   integer     NOT NULL DEFAULT 0,
  last_accessed_at timestamptz
);

-- HNSW index for vector similarity
CREATE INDEX IF NOT EXISTS idx_node_embedding ON node
  USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64);

-- GIN index for full-text search
CREATE INDEX IF NOT EXISTS idx_node_search_text ON node USING gin (search_text);

-- Partial index for forgetting curve candidates
CREATE INDEX IF NOT EXISTS idx_node_last_accessed ON node (last_accessed_at)
  WHERE last_accessed_at IS NOT NULL;

-- Trigger: auto-populate search_text from body
CREATE OR REPLACE TRIGGER trg_node_search_text
  BEFORE INSERT OR UPDATE OF body ON node
  FOR EACH ROW EXECUTE FUNCTION update_search_text();

-- Trigger: auto-update updated_at
CREATE OR REPLACE TRIGGER trg_node_updated_at
  BEFORE UPDATE ON node
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### Knowledge Graph: `edge`

```sql
CREATE TABLE IF NOT EXISTS edge (
  id         integer     GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  source_id  integer     NOT NULL REFERENCES node(id) ON DELETE CASCADE,
  target_id  integer     NOT NULL REFERENCES node(id) ON DELETE CASCADE,
  label      text        NOT NULL,  -- extensible: 'related_to', 'caused_by', ...
  weight     real        NOT NULL DEFAULT 1.0
             CHECK (weight >= 0.0 AND weight <= 5.0),
  valid_from timestamptz NOT NULL DEFAULT now(),
  valid_to   timestamptz,           -- NULL = currently active
  created_at timestamptz NOT NULL DEFAULT now()
);

-- Active edges only (most queries filter on this)
CREATE INDEX IF NOT EXISTS idx_edge_active ON edge (source_id, target_id)
  WHERE valid_to IS NULL;

-- FK indexes
CREATE INDEX IF NOT EXISTS idx_edge_source ON edge (source_id);
CREATE INDEX IF NOT EXISTS idx_edge_target ON edge (target_id);
```

### Episodic Memory: `episode`

```sql
CREATE TABLE IF NOT EXISTS episode (
  id            integer     GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id    text        NOT NULL,
  role          text        NOT NULL
                CHECK (role IN ('user', 'assistant')),
  body          text        NOT NULL,
  embedding     vector(768),
  search_text   tsvector,
  superseded_by integer     REFERENCES episode(id) ON DELETE SET NULL,
  created_at    timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_episode_session ON episode (session_id);
CREATE INDEX IF NOT EXISTS idx_episode_embedding ON episode
  USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64);
CREATE INDEX IF NOT EXISTS idx_episode_search_text ON episode USING gin (search_text);
CREATE INDEX IF NOT EXISTS idx_episode_superseded ON episode (superseded_by)
  WHERE superseded_by IS NOT NULL;

-- Trigger: auto-populate search_text from body (reuses update_search_text)
CREATE OR REPLACE TRIGGER trg_episode_search_text
  BEFORE INSERT OR UPDATE OF body ON episode
  FOR EACH ROW EXECUTE FUNCTION update_search_text();
```

`superseded_by` uses `ON DELETE SET NULL`: if a consolidated episode is deleted, the originals
become visible again rather than being cascade-deleted.

### Episode-Node Cross-Reference: `episode_node`

```sql
CREATE TABLE IF NOT EXISTS episode_node (
  episode_id integer NOT NULL REFERENCES episode(id) ON DELETE CASCADE,
  node_id    integer NOT NULL REFERENCES node(id) ON DELETE CASCADE,
  created_at timestamptz NOT NULL DEFAULT now(),
  PRIMARY KEY (episode_id, node_id)

CREATE INDEX IF NOT EXISTS idx_episode_node_node ON episode_node (node_id);
```

### Core Memory: `core_memory`

```sql
CREATE TABLE IF NOT EXISTS core_memory (

  ('persona', '{}'),

```

### Procedural Memory: `skill`

```sql
CREATE TABLE IF NOT EXISTS skill (
  id                integer     GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name              text        NOT NULL,
  trigger_context   text        NOT NULL,
  trigger_embedding vector(768),
  strategy          text        NOT NULL,
  success_count     integer     NOT NULL DEFAULT 0,
  attempt_count     integer     NOT NULL DEFAULT 0,
  success_rate      real        GENERATED ALWAYS AS (
    CASE WHEN attempt_count = 0 THEN 0.0
         ELSE success_count::real / attempt_count
    END
  ) STORED,
  version           integer     NOT NULL DEFAULT 1,
  parent_id         integer     REFERENCES skill(id) ON DELETE SET NULL,
  promoted_at       timestamptz,
  domain            text[]      NOT NULL DEFAULT '{}',
  created_at        timestamptz NOT NULL DEFAULT now(),
  updated_at        timestamptz NOT NULL DEFAULT now()
);

-- HNSW index for trigger similarity search
CREATE INDEX IF NOT EXISTS idx_skill_trigger_embedding ON skill
  USING hnsw (trigger_embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64);

-- Active (non-promoted) skills for retrieval
CREATE INDEX IF NOT EXISTS idx_skill_active ON skill (updated_at DESC)
  WHERE promoted_at IS NULL;

-- Version lineage
CREATE INDEX IF NOT EXISTS idx_skill_parent ON skill (parent_id)
  WHERE parent_id IS NOT NULL;

-- Trigger: auto-update updated_at
CREATE OR REPLACE TRIGGER trg_skill_updated_at
  BEFORE UPDATE ON skill
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

`success_rate` is a generated column — always consistent with `success_count / attempt_count`.
`parent_id` tracks version lineage: when a skill is refined, the new version references the old one.
`promoted_at` marks skills that have been elevated to the persona.

## Risks

**Low risk.** The schema is well-specified in `docs/foundation.md` and `.claude/rules/memory.md`.
The `update_search_text()` trigger function works for both `node` and `episode` since both have a
`body` column.

Note: the HNSW index on `embedding` columns will only be effective after embeddings are populated
(Phase 5). Until then, the index exists but is empty, which is fine.
