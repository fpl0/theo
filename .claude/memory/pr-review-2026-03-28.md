## Systemic Patterns Observed

1. **Branch staleness**: PR #26 was branched before conversation/ refactoring. PRs that touch core modules risk diverging from main quickly. Consider shorter-lived branches or more frequent rebases.

2. **Broad exception suppression**: `contextlib.suppress(Exception)` used for optional integrations. This hides real bugs. Pattern should be narrowed to specific expected exceptions (ImportError for optional modules).

3. **FOR UPDATE SKIP LOCKED misuse**: The initial implementation used `FOR UPDATE SKIP LOCKED` outside a transaction (autocommit mode), making the lock a no-op. This is a subtle concurrency bug that passes all unit tests since they mock the pool. Integration tests against a real database would catch this.

4. **Migration numbering without main sync**: Both PRs chose migration numbers without checking current main state. A pre-commit hook or CI check that verifies migration numbers don't collide with main would prevent this class of error.

5. **Observable gauges in async contexts**: The intent module tried to create an OTEL observable gauge with a sync callback that couldn't await — resulting in a permanently-zero metric. Async-native codebases need async-compatible metric patterns.

## Updated Systemic Patterns

6. **Caller/callee contract mismatch**: The diminishing-returns bug was a classic case where unit tests for the pure function pass, but the caller passes semantically wrong arguments. Integration tests that mock at the boundary (embedder, DB) but let internal functions run would have caught this. Pattern seen in both PR #26 (conversation.py stale imports) and PR #31 (node list accumulation).

7. **Embedding recomputation**: The metacognition monitor re-embeds all prior phase outputs on every call (15 total for 5 phases). Caching embeddings at the call site would reduce this to 5. Worth tracking as a pattern — any module that calls embedder in a loop should consider caching.
