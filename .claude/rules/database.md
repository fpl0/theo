# Database conventions

- **postgres.js tagged templates.** No ORM. No query builder. All queries
  use `` sql`...` `` tagged templates which handle parameterization
  automatically.
- **Connection config**: postgres.js options object with `max`,
  `idle_timeout`, `connect_timeout`. Connection string from env via zod
  config.
- **pgvector**: extension created in migration 0001. Vectors stored as
  `vector(768)` columns. Cosine distance via `<=>`.
- **Migrations** are `.sql` files named `NNNN_description.sql`.
  Forward-only. No down migrations. Each runs in its own transaction.
- Use `IF NOT EXISTS` / `IF EXISTS` guards for DDL.
- Use `timestamptz` for all timestamps, never `timestamp`.
- Use `GENERATED BY DEFAULT AS IDENTITY` for auto-increment PKs (allows explicit IDs during event replay).
- Add `created_at timestamptz NOT NULL DEFAULT now()` to all tables.
- Add `updated_at` + trigger where rows will be modified.
- Add indexes for foreign keys (PostgreSQL does NOT auto-create them).
- Table and column names are `snake_case`. No quoted identifiers.
- Use `text` over `varchar(n)` unless there is a genuine length constraint.
